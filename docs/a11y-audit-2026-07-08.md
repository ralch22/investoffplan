## Priority Fixes (top 10, by impact ÷ effort)

1. **Add labels + autocomplete to the three placeholder-only forms** (U1 — `contact-form.tsx`, `newsletter-form.tsx`, `mortgage-preapproval-form.tsx`). Highest user impact; the labelled pattern already exists in `brochure-modal.tsx`. These are the site's revenue forms.
2. **Fix newsletter error color** (P1 — `newsletter-form.tsx:112,131`): swap `text-red-400` → `.iop-field-error`. One-line, 2.77→4.81:1.
3. **Make brochure modal, mobile nav, and mobile filter sheet real dialogs** (O1/O2): move to `<dialog>.showModal()` as done in `media-gallery-lightbox.tsx` — gets focus trap + Escape + restore in one refactor.
4. **Name the lightbox thumbnail buttons** (R1 — `media-gallery-lightbox.tsx:141`): one `aria-label`, removes ~89 unnamed controls from the SERP.
5. **`<main id="main-content">` in page-shell** (R2 — `page-shell.tsx:36`): one-tag change fixes the missing landmark on home/contact/ar and makes the skip link land on a landmark.
6. **Associate + announce form errors** (U2/U3): copy contact-form's `aria-describedby` pattern into newsletter + pre-approval; add `role="alert"`/`role="status"` to error/success blocks across all four forms.
7. **Fix active-nav pill contrast** (P2/P3 — `site-header.tsx:103`, `mobile-nav.tsx:71`, `newsletter-form.tsx:87`): use `--brand-dark` on the 12%-red fill.
8. **Disambiguate compare checkboxes** (O3 — `compare-bar.tsx:106`): interpolate project name into `aria-label`.
9. **Guarantee header contrast over hero photos** (P4): strengthen the top stop of `--hero-overlay` or add a header scrim — protects against any future CMS image.
10. **Strip empty headings in `sanitizeProjectHtml` + darken `--text-muted-light` and input borders** (P5/P6/P7): token-level changes in `globals.css:10,20` + one sanitizer rule fix every instance at once.
